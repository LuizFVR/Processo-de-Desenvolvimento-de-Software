Análise e Projeto

Descrição do 2º ciclo do processo de análise e projeto, abordando as etapas principais e o detalhamento das atividades, papéis e artefatos.

Atividades

Levantamento de Requisitos

  • Finalidade: entender o que o cliente quer
  • Artefatos de Entrada: documentação inicial - DI
  • Artefatos de Saída: documento de requisitos - DR
  • Papéis: analista de requisitos - AR
  • Ferramentas: word, entrevista
  • Passos:
    1- Marcar uma entrevista com o cliente
    2- Entender o que o cliente quer
    3- Registrar os dados no Word

Análise de Requisitos

  • Finalidade: trabalhar com os dados obtidos no levantamento de requisitos
  • Artefatos de Entrada: documento de requisitos - DR
  • Artefatos de Saída: diagrama de caso de uso - DCU
  • Papéis: analista de requisitos - AR
  • Ferramentas:
  • Passos:
    1-Pegar os dados registrados
    2- Organizá-los
    3- Fazer um Diagrama de Caso de Uso

Desenho de Arquitetura

  • Finalidade: estruturar a arquitetura do projeto
  • Artefatos de Entrada: diagrama de caso de uso - DCU
  • Artefatos de Saída: diagrama de arquitetura - DA
  • Papéis: arquiteto de software - AS
  • Ferramentas: miro
  • Passos:
    1- Pegar o Diagrama de Caso de Uso
    2- Abrir o miro
    3- Se basear no diagrama para desenvolver a arquitetura

Prototipagem

  • Finalidade: fazer um protótipo para ver se o cliente teve os requisitos atendidos
  • Artefatos de Entrada:diagrama de arquitetura - DA
  • Artefatos de Saída: protótipo - PR
  • Papéis:desenvolvedor - DEV
  • Ferramentas: vscode
  • Passos:
    1-Escolher o Software
    2-Desenvolver com base nos requisitos
    3-Mostrar ao cliente

Planejamento de Interações

  • Finalidade: usa o feedback do cliente para planejar os próximos passos
  • Artefatos de Entrada: protótipo - PR
  • Artefatos de Saída: plano de sprint - PS
  • Papéis: product owner - PO
  • Ferramentas: word
  • Passos:
    1-Resume o feedback do cliente
    2-Abre o Word
    3-Anota os dados
    4-Planeja os próximos passos

Papéis

Analista de Requisitos

  • Atribuições: analisar os requisitos
  • Atividades: levantamento de requisitos e análise de requisitos
  • Escolaridade: graduação em TI
  • Conhecimento: engenharia de software
  • Cargos: analista de requisitos

Arquiteto de Software

  • Atribuições: responsável pela arquitetura do software
  • Atividades: desenho da arquitetura
  • Escolaridade: graduação em TI
  • Conhecimento: arquitetura de software
  • Cargos: arquiteto de software

Desenvolvedor

  • Atribuições: desenvolver e testar um software
  • Atividades: prototipagem
  • Escolaridade: graduação em TI
  • Conhecimento: linguagens de programação, testes unitários e banco de dados
  • Cargos: desenvolvedor de software

Product Owner

  • Atribuições: definir objetivos, estabelecer o cronograma e evitar riscos
  • Atividades: planejamento de interações
  • Escolaridade: graduação em TI
  • Conhecimento: gestão de produtos, metodologias ágeis, análise de dados e métricas
  • Cargos: product owner

Artefatos

Documentação Inicial

  • Descrição: documentação necessária para começar o levantamento de requisitos
  • Atividades: levantamento de requisitos
  • Papel: analista de requisitos
  • Ferramentas: word

Documento de Requisitos

  • Descrição: documento que especifica os requisitos
  • Atividades: análise de requisitos
  • Papel: analista de requisitos
  • Ferramentas: word

Diagramas de caso de uso

  • Descrição: representa visualmente as interações entre usuários e o sistema
  • Atividades: desenho de arquitetura
  • Papel: arquiteto de software
  • Ferramentas: draw.io

Diagrama de arquitetura

  • Descrição: representação visual que descreve a estrutura e a organização de um sistema ou projeto
  • Atividades: prototipagem
  • Papel:desenvolvedor
  • Ferramentas: draw.io

Protótipo

  • Descrição: amostra do modelo do produto
  • Atividades: prototipagem
  • Papel: desenvolvedor
  • Ferramentas: IDE, linguagens de programação

Plano de Sprint

  • Descrição: define o que pode ser entregue no sprint e como será feito
  • Atividades: planejamento de interações
  • Papel: product owner
  • Ferramentas: trello
