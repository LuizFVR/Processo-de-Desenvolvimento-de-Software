Validação e Verificação

Descrição do 2º ciclo do processo de validação e verificação, abordando as etapas principais e o detalhamento das atividades, papéis e artefatos.

Atividades

Revisão de Requisitos

  • Finalidade: garantir que todos os requisitos atendam às necessidades dos clientes
  • Artefatos de Entrada: documento de requisitos - DR
  • Artefatos de Saída: documento de design - DD
  • Papéis: analista de requisitos - AR
  • Ferramentas: word
  • Passos:
    1- Realizar a leitura e análise do documento de requisitos
    2- Conduzir uma reunião com os stakeholders e a equipe
    3- Atualizar o documento de requisitos

Revisão de Design

  • Finalidade: verificar se o design atende aos requisitos
  • Artefatos de Entrada: documento de design - DD
  • Artefatos de Saída: relatório de revisão - RR
  • Papéis: arquiteto de software - AS
  • Ferramentas: ferramentas de modelagem
  • Passos:
    1- Revisar o design conforme os requisitos
    2- Registrar em um relatório de revisão

Teste de Unidade

  • Finalidade: validar o funcionamento de pequenas unidades de código
  • Artefatos de Entrada: código fonte - CF
  • Artefatos de Saída: relatório de testes de unidade - RTU
  • Papéis: desenvolvedor - DEV
  • Ferramentas: frameworks de teste
  • Passos:
    1- Escrever e executar os testes de unidade
    2- Corrigir os erros encontrados
    3- Gerar um relatório de testes de unidade com as descobertas

Teste de Integração

  • Finalidade: validar a interação entre diferentes módulos ou sistemas para garantir que eles funcionem juntos
  • Artefatos de Entrada: relatório de testes de unidade - RTU
  • Artefatos de Saída: relatório de testes de integração - RTI
  • Papéis: engenheiro de testes - ET
  • Ferramentas: jenkins
  • Passos:
    1- Configurar o ambiente
    2- Executar os testes
    3- Documentar os resultados

Teste de Aceitação

  • Finalidade: validar se o software atende aos requisitos e expectativas dos clientes
  • Artefatos de Entrada: relatório de testes de integração - RTI
  • Artefatos de Saída: relatório de testes de aceitação - RTA
  • Papéis: representante de usuário - RU
  • Ferramentas: zephyr e google forms
  • Passos:
    1- Revisar os casos de teste de aceitação
    2- Configurar o ambiente de teste
    3- Executar os testes
    4- Gerar um relatório e obter aprovação do cliente

Papéis

Analista de Requisitos

  • Atribuições: analisar os requisitos
  • Atividades: revisão de requisitos
  • Escolaridade: graduação em TI
  • Conhecimento: engenharia de software
  • Cargos: analista de requisitos

Arquiteto de Software

  • Atribuições: responsável pela arquitetura do software
  • Atividades: revisão de design
  • Escolaridade: graduação em TI
  • Conhecimento: arquitetura de software
  • Cargos: arquiteto de software

Desenvolvedor

  • Atribuições: desenvolver e testar um software
  • Atividades: teste de unidade
  • Escolaridade: graduação em TI
  • Conhecimento: linguagens de programação, testes unitários e banco de dados
  • Cargos: desenvolvedor de software

Engenheiro de Testes

  • Atribuições: criar e manter casos de teste e scripts para os testes de integração.
  • Atividades: teste de integração
  • Escolaridade: graduação em TI
  • Conhecimento: conhecimento em metodologias de teste de software, ferramentas de automação de testes
  • Cargos: engenheiro de testes

Representante de Usuário

  • Atribuições: assegura que o sistema atende às necessidades especificadas pelo cliente
  • Atividades: teste de aceitação
  • Escolaridade: graduação em TI
  • Conhecimento: familiaridade com os processos e necessidades da organização
  • Cargos: representante de usuário

Artefatos

Documento de Requisitos

  • Descrição: documento que descreve as funcionalidades
  • Atividades: revisão de requisitos
  • Papel: analista de requisitos
  • Ferramentas: word

Documento de Design

  • Descrição: documento que contém o design arquitetural e detalhado do sistemas
  • Atividades: revisão de requisitos e revisão de design
  • Papel: analista de requisitos e arquitetura de software
  • Ferramentas: ferramentas de modelagem

Relatório de Testes de Unidade

  • Descrição: registra os resultados dos testes de unidade realizados sobre funções e métodos específicos no código
  • Atividades: revisão de design
  • Papel: arquiteto de software
  • Ferramentas: frameworks de teste

Código Fonte

  • Descrição: origem de um programa de computador
  • Atividades: teste de unidade
  • Papel: desenvolvimento
  • Ferramentas: IDEs

Relatório de Teste de Unidade

  • Descrição: é um processo de verificação que avalia a menor parte testável do software
  • Atividades: teste de unidade e teste de integração
  • Papel: desenvolvimento e engenheiro de testes
  • Ferramentas: IDEs

Relatório de Teste de Integração

  • Descrição: avalia como diferentes módulos ou componentes do software funcionam em conjunto
  • Atividades: teste de integração, teste de aceitação
  • Papel: engenheiro de testes e representante de usuário
  • Ferramentas: IDEs

Relatório de Teste de Aceitação

  • Descrição: realizado para validar se o software atende aos requisitos e expectativas do cliente ou usuário final
  • Atividades: teste de aceitação
  • Papel: representante de usuário
  • Ferramentas: Cypress,
